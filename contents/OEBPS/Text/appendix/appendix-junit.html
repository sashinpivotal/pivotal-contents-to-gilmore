JUnit and IDEs

Purpose

This appendix will demonstrate IDE features for JUnit test support in STS/Eclipse and IntelliJ IDEA.

Implementing and running tests in a Java development environment requires multiple sets

Generate a JUnit Test in IDE

STS or Eclipse

  • Right click on the class under test in the tree view.

  • Select New >> JUnit Test Case menu option.

  • Eclipse will show a JUnit test dialog:

    You will notice a few things:

    • In this course we are using JUnit 5, it will be selected by default given it is in the classpath for our projects.

    • It will default the test case to the same package level as the class under test.

    • The test class will be named as the Test.

    • You may change any of the defaults if you wish.

  • You may select a boiler plate any of the method stubs as appropriate:

    • setUp()
    • tearDown()
    • setUpBeforeClass()
    • tearDownAfterClass()

  • Click Next button.

  • Eclipse will show a dialog list of methods from the class user test that you may want to build tests for.

    Choose the ones you want to build test case.

  • Click Finish button.

  • Review the following generated test class.

    You will notice a couple of things:

    • You will notice that that the selected setUp() method from the wizard automatically translates to JUnit 5.

    • Test cases implemented will be setup by default to fail. You must implement to pass.

IDEA IntelliJ

In IntelliJ, you may generate a test class for a given class under test by doing the following:

  1. Navigate to the class under test. Open it in IntelliJ editor, and make sure the mouse focus is in the associated class edit window.

  2. Do one of either to launch JUnit test creation option:

    • Use keyboard shortcut Cmd-Shift-T on Mac OSX, or Ctrl+Shift-T on Linux or Windows.

    • Select the Navigate >> Test menu option.

  3. The test creation option will appear as follows:

  4. Select Create New Test

  5. IntelliJ will show a JUnit test dialog:

    You will notice a few things:

    • In this course we are using JUnit 5, it will be selected by default given it is in the classpath for our projects.

    • It will default the test case to the same package level as the class under test.

    • The test class will be named as the Test.

    • You may change any of the defaults if you wish.

  • You may select a boiler plate any of the method stubs as appropriate:

    • setUp()/@Before
    • tearDown()/@After

  • IntelliJ will show a dialog list of methods from the class user test that you may want to build tests for.

    Choose the ones you want to build test case.

  • Click OK button.

  • Review the following generated test class.

    You will notice a couple of things:

    • You will notice that that the selected setUp() method from the wizard automatically translates to JUnit 5.

    • Test cases implemented will be empty. You must implement them.

Run a JUnit Test from IDE

STS or Eclipse

From the test class editor, or when selecting the test class in the project tree view, you may use either of the following:

  • Right-click on the test class name, select Run As >> JUnit Test.

  • Use keyboard shortcut Opt-Cmd-X,T on Mac OSX, or Alt+Shift+X,T on Linux or Windows.

Eclipse will render visual tool window showing the status of your test execution. You may exand the test case node for details:

IDEA IntelliJ

From the test class editor, or when selecting the test class in the project tree view, you may use either of the following:

  • Right-click on the test class name, select Run .

  • Use keyboard shortcut Ctrl-Shift-R on Mac OSX, or Ctrl+Shift+R on Linux or Windows.

IntelliJ will render visual tool window showing the status of your test execution:
