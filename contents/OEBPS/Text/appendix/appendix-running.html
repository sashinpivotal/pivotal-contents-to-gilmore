Running Spring Boot Applications

Purpose

This document describes options for running Maven or Gradle-based Spring Boot applications, both at the command line and within an IDE.

Command line

Command Line via Java

The simplest way to run a Spring Boot app is directly via Java through a packaged Spring Boot jar file.

java -jar 32-jdbc-autoconfig-solution-5.0.c.RELEASE.jar

Command Line via Maven

You may run from the multi-module root. Under the lab directory, run with the Maven wrapper running as follows:

./mvnw -pl 00-rewards-common -pl 01-rewards-db clean install (need to be done just once)
./mvnw -pl <lab project module> clean spring-boot:run

Or you can run from the project directory:

../mvnw clean spring-boot:run

Command Line via Gradle

You may run from the multi-module root. Under the lab directory, run with the Gradle wrapper running as follows:

./gradlew <lab project module>:clean <lab project module>:bootRun

Or you can run from the project directory:

../gradlew clean bootRun

STS/Eclipse

You have 4 options:

  • Through Maven(M2) Plugin
  • Through Gradle Plugin
  • Through Run Configuration
  • Through STS Spring Boot Dashboard

The first 3 options are handled with Eclipse Run Configuration feature.

Through Maven(M2) Plugin

Configure a run configuration to execute spring-boot:run goal for Maven for your project:

  • Navigate to Run >> Run Configurations top-level menu option.

  • Select Maven Build.

  • Click on New Configuration + button (located top-left corner).

  • Select the appropriate workspace (project).

  • Click OK.

  • Enter spring-boot:run in the Goals text box, and a configuration name.

  • Click Apply.

  • Click Run.

Through Gradle Plugin

Configure a run configuration to execute bootRun task for Gradle for your project:

  • Navigate to Run >> Run Configurations top-level menu option.

  • Select Gradle Project.

  • Click on New Configuration + button (located at top-left corner).

  • Select the appropriate workspace (project).

  • Click OK.

  • Enter bootRun in the Gradle Tasks text box, and a configuration name.

  • Click Apply.

  • Click Run.

Through Run Configuration

In STS/Eclipse, you can use the standard way of running applications through the IDE runner: The Run Configuration

  • You may choose:

    • Right click on a class with main() method and select Run As >> Run Configurations.

    • Click on a class with main() method and navigate to the Run >> Run Configurations top-level menu.

    • Without a class in focus, select Run >> Run Configurations and add the appropriate class to a Spring Boot App type.

  • If you started with a class in focus, the Project and main type will be filled for you.

  • If you started from top-level menu option without a class in focus, you will need to select the project and main type.

  • Click on the Run button.

  • You may also use the Run Configuration to debug.

Spring Boot Dashboard

When you import working sets in STS (or Eclipse with STS plugin), it will look for Spring Boot context.

If it finds Spring Boot application it will show it in Spring Boot Dashboard.

You can run by right clicking the appropriate project and select (Re)start.

You will see the Spring Boot console output in the Console window.

IntelliJ

Similar to STS/Eclipse, you have 4 options to run Spring Boot app:

  • Through Maven Tools
  • Through Gradle Tools
  • Through Run Configuration
  • Through Spring Boot Dashboard (Ultimate Edition only)

Through Maven Tools

  • Expand the target module in the Maven toolbox.

  • Expand the Plugins >> spring-boot plugin.

  • Double click on the run goal, or right click on the run goal and select Run Maven Build.

Note: The current Spring Boot course multi-module project structure does not include a build configuration for the spring-boot-plugin. Because of this you will not see the spring-boot plugin in the IntelliJ Maven Projects view.

Through Gradle Tools

  • Expand the target module in the Gradle toolbox.

  • Expand the Tasks >> application.

  • Right click on the bootRun task and select Run lab:.

Through Run Configuration

IntelliJ has a run configuration similar to Eclipse.

You can configure to run a Spring Boot application in a couple of ways:

  • Right click on the Spring Boot application class, select either Run or Debug.

    You may also use Ctrl+Shift+R shortcut to run it.

    Running it this way will automatically create a run configuration for it, or use the same configuration created if you selected to run the Spring Boot Dashboard when you imported your project.

  • The console output will show in the Console tool window at the IDE footer.

    You will notice if you previously chose to run the Spring Boot Dashboard, you will see the status of the Spring Boot app to Running.

  • You may also configure a runner manually through the Run configuration toolbar option.

  • Click the pulldown and select Edit Configurations.

  • Select Spring Boot type of runner and click the + option to add a new configuration.

  • Enter a configuration name and select a main class.

  • You might use the manual run config to set up a second instance of a Spring Boot application on a different port. In this case you must set a port configuration in its environment.

  • Expand the Environment option:

  • Click on the elipse (...) button next to Environment variables text box.

  • Click the + button and add environment variables key/value pair.

  • Click OK button.

  • Click OK button.

Through Spring Boot Dashboard

Running the IntelliJ Spring Dashboard is a bit different than running STS. It is only enabled with the combination of two specfic conditions:

  • There is more than one Spring Boot application in a multi-module project.

  • You explicitly select the option to run the Dashboard when running the project.

The Spring Dashboard works by setting up a Run Configuration for each Spring Boot application detected in your multi-module build.

You may override or edit the associated Run Configuration as you would any other, see Run Configuration for more details.

Steps to configure a multi-module project with Spring Dashboard:

  • Open IntelliJ, or post the creation dialog.

  • Click the Import Project option.

  • Select the directory root of the multi-module project you want to import.

  • Import the multi-module project as a Maven external project type.

  • Click Next button.

  • In the Import Project Dialog, select the following options:

    • Search for projects recursively

    • Optional - Import Maven projects automatically

      If you do not select this, than any changes in your Maven any and all project POM changes will require a manual refresh.

  • Click Next.

  • Select all of the projects you want to import in the second Import Projects dialog.

  • Click Next.

  • Continue with the standard IntelliJ Java project creation steps.

  • When your Spring Boot multi-module project loads the first time you will be prompted with the Run Dashboard dialog:

  • Click on it.

  • Click on the Show run configurations in Run Dashboard option.

    The Run Dashboard tool windows will show by default in the footer:

  • If you close the Run Dashboard and new to view it later, you may select it from the View >> Tool Windows >> Run Dashboard option.

Running a Spring Boot application from the Run Dashboard:

  • Right click on the app you want to run, select Run or Debug as appropriate.

  • The Spring Boot app will run in the Console tool window in the IDE footer:

Running an Integration Test

Ideally you run the integration tests with the build tool test lifecycle command, or directly from an IDE.

Command Line

You may run from the multi-module root. Under the labs directory run with the Maven or Gradle wrapper running as follows:

./mvnw -pl <lab project module> clean test 

./gradlew <lab project module>:clean <lab project module>:test

or indirectly as part of build:

./mvnw -pl <lab project module> clean package 

./gradlew <lab project module>:clean <lab project module>:build

Or you can run from the project directory:

../mvnw clean test

../gradlew clean test

or indirectly as part of build:

../mvnw clean package

../gradlew clean build

STS/Eclipse

You have 3 options:

  • Through Maven(M2) Plugin
  • Through Gradle Plugin
  • Through Run Configuration

Through Maven(M2) Plugin

Configure a run configuration to execute test goal for your project:

  • Navigate to Run >> Run Configurations top-level menu option.

  • Select Maven Build.

  • Click on New Configuration + button.

  • Select the appropriate workspace (project).

  • Click OK.

  • Enter test in the Goals text box, and a configuration name.

  • Click Apply.

  • Click Run.

  • Output will show in the Console.

Through Gradle Plugin

Configure a run configuration to execute test task for Gradle for your project:

  • Navigate to Run >> Run Configurations menu option.

  • Select Gradle Project.

  • Click on New Configuration + button.

  • Select the appropriate workspace (project).

  • Click OK.

  • Enter test in the Gradle Tasks text box, and a configuration name.

  • Click Apply.

  • Click Run.

  • Output will show in the Console.

Run Configuration

In STS/Eclipse, you can use the standard way of running applications through the IDE runner: The Run Configuration

  • You may either:

    • Right click on a Test class and select Run As >> Run Configurations

    • Click on a test class and navigate to Run >> Run Configurations menu.

  • Select JUnit type.

  • Click on New Configuration + button.

  • Select the project.

  • Enter the run config name and select JUnit 5 as the Test Runner type.

  • Search for the Test

  • Click OK.

  • Search for the Test Methods.

  • Select (all methods).

  • Click OK.

  • Click Apply.

  • Click Run.

  • Output will show in the Console.

  • You may also use the Run Configuration to debug.

IntelliJ

Similar to STS/Eclipse, you have 3 options:

  • Through Maven Tools
  • Through Gradle Tools
  • Through Run Configuration

Through Maven Tools

If you want to run all tests in your project:

  • In the Maven Projects tool view, expand the module where you want to run your tests.

  • Expand the Lifecycle node.

  • Double click on the test goal, or right click on the test goal and select Run Maven Build.

Through Gradle Tools

  • Expand the target module in the Gradle toolbox.

  • Expand the Tasks >> verification.

  • Right click on the test task and select Run lab:.

Through Run Configuration

Similar to STS, you can run a Test through a run configuration.

You have a couple of options.

  • Right click on your test, select either Run or Debug. You can use the Cntrl+Shift+R and Cntrl+Shift+D shortcuts respectively.

  • The console output shows in the Console tool view at the IDE footer.
